LiteLLM

Boss Agent

🛠

Worker Agents

Execute tasks with reflection + memory

👀

Monitors health, restarts frozen containers

Optional profiles: Prometheus + Grafana (monitoring), OpenBao (secrets), Caddy (reverse proxy + auto-TLS), Open WebUI (chat), Watchtower (auto-update)

Socket Architecture

We provide sockets — standard interfaces that any framework can connect to. Swap components without touching code.

LiteLLM OpenAI-compatible API — plug in LangGraph, CrewAI, AutoGen, any OpenAI client
Redis queues tasks:{agent}, results:boss — custom orchestrators, n8n, any queue consumer
PostgreSQL agent_tasks, agent_budgets, agent_autonomy — Braintrust, Promptfoo, custom dashboards
ChromaDB Vector store API — LlamaIndex, Haystack, any RAG pipeline
Docker network superteaim-net bridge — any new container joins and connects
.env config All behavior configurable — swap components without touching code

Built-in Intelligence

No external framework dependencies. These capabilities are built into the agents.

Reflection

Workers self-check output before returning. Configurable via ENABLE_REFLECTION.

Memory

Workers query past similar tasks from PostgreSQL before processing.

Planning

Boss breaks complex requests into parallel + sequential subtasks.

Autonomy Levels

Agents earn trust through track record. L0 supervised to L2 autonomous.

Token Budgets

Data Sovereignty

Every request is classified before routing. Your sensitive data never leaves your network.

VAULT

  • Local storage only
  • Processed by local models only
  • Never leaves your network
  • HR, finance, credentials, PII

LIBRARY

  • Can be stored anywhere
  • Any model including cloud APIs
  • Public docs, research, open data
  • Non-sensitive by classification

If unsure, the Boss Agent defaults to VAULT — local only.

Quick Start

Up and running in under a minute.

# Clone and configure
$ git clone https://github.com/livelomallorca/superteaim.git
$ cd superteaim
$ cp config/.env.example .env
$ nano .env # set your INFERENCE_URL

# Launch
$ docker compose up -d

# Or use the interactive wizard
$ ./scripts/setup.sh
Requirements: Docker + Compose v2, an inference endpoint (Ollama, vLLM, or cloud API), 4GB+ RAM. No GPU needed on the server.
